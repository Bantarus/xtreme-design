Storybook is the composition canvas for chapter 2. Where chapter 1 (the gallery) shows one component at a time in the active palette, chapter 2 shows full pages and sections — landing pages, dashboards, settings screens, wizards — composed from the shadcn components and re-themed instantly through the version picker.

## How the version-aware decorator works

`apps/storybook/.storybook/decorators/ActiveStylesheet.tsx` is a React decorator wrapped globally. Per story render, `useEffect` reads `context.globals.designVersion` and:

- `"active"` → removes any prior injected `<link>`. The preview falls back to `preview.css`'s statically imported `apps/gallery/app/tokens.active.css`, which is the live current design system.
- `"v1"`, `"v2"`, `"base"`, etc. → injects `<link rel="stylesheet" href="/versions/<id>/tokens.css" data-dsx-version="<id>">` at the end of `<head>`. CSS cascade order makes the later sheet's `:root { --color-* }` block win over the earlier one. Swap latency: ~30 ms (link load + paint) in dev.

The toolbar paintbrush surfaces every entry from `.storybook/versions.json`, which `scripts/mirror-storybook-versions.mjs` regenerates whenever a snapshot is created. Vite HMR picks up the file change so new versions appear in the toolbar without restarting Storybook.

## How a new page reference gets added

The corpus in `.dsx/page-references/` is the source the `/page` and `/section` commands score against. Each file is a working CSF3 story with a leading JSDoc block:

```ts
/**
 * @genre landing | dashboard | settings | pricing | auth | empty | error | wizard
 * @keywords saas, marketing, hero, indie, developer
 * @description One-sentence purpose statement.
 */
```

To add a new reference:

1. Create `.dsx/page-references/<descriptive-slug>.stories.tsx`.
2. Open with the JSDoc block. Pick a genre from the existing set (or invent a new one — the matcher is purely keyword-based, so genre is one more keyword in the haystack).
3. Author a single CSF3 default + `Default` story with realistic content. Use `parameters: { layout: 'fullscreen' }` for pages, `centered` for sections. Import from `@gallery/ui/*`.
4. The next `node scripts/find-page-reference.mjs "<test brief>"` will include it. No registration needed.

These files live outside Storybook's `stories` glob, so they don't render in the running Storybook. They're a corpus, not stories.

- **Storybook visual regression.** Gallery visual regression continues via Playwright; Storybook stays out of the snapshot suite until a future chapter.

## When something fails

| Symptom | Likely cause | Fix |
|---|---|---|
| Toolbar paintbrush is empty | mirror hasn't run | `pnpm pipeline` once |
| Switching versions doesn't change the palette | network panel shows 404 on `/versions/<id>/tokens.css` | `pnpm pipeline` regenerates mirror |
| Switching versions changes one var but not its Tailwind utility | utility is being purged because the @source globs don't reach the file | extend `@source` in `apps/storybook/.storybook/preview.css` |
| Storybook restarts on every DESIGN.md save | watcher and Storybook fighting over `tokens.active.css` | check that the mirror writes happen AFTER snapshot — pipeline order is enforced |
| New page reference doesn't show up in the corpus | filename doesn't end `.stories.tsx`, or JSDoc block isn't the very first thing | both required by `scripts/find-page-reference.mjs` |
